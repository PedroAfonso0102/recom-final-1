# RECOM — Guia de tom, copywriting, ghostwriter e raciocínio de escrita

**Projeto:** RECOM Metal Duro  
**Data:** 2026-04-28  
**Uso:** guia para humanos, LLMs, redatores, designer, coder e CMS.

---

## 1. Tese de voz da RECOM

A RECOM deve soar como uma empresa B2B tradicional, local, confiável e prática. A escrita precisa transmitir experiência e seriedade sem parecer antiga, burocrática ou distante.

A marca não deve tentar soar como fabricante global nem como loja de varejo. O papel correto é o de distribuidor B2B que ajuda o cliente industrial a encontrar fornecedores, catálogos, processos e contato comercial com menos atrito.

### Fórmula central

```txt
RECOM = tradição comprovável + atendimento próximo + fornecedores reconhecidos + clareza para chegar ao catálogo/contato certo
```

## 2. Diagnóstico do tom atual do site legado

### 2.1. Como o site atual soa

- Institucional antigo.
- Técnico e didático.
- Focado em metal duro, processos e produtos Mitsubishi.
- Mais informativo do que comercial.
- Pouco persuasivo.
- Pouco orientado a CTA.
- Confiável pela substância, mas datado pela forma.

### 2.2. O que há de bom

- A escrita comunica existência real.
- Há histórico e localização.
- Há relação com marcas/fornecedores.
- Há vocabulário técnico legítimo.
- Há foco em segurança, usinagem e processos.
- O contato aparece com clareza.

### 2.3. O que precisa mudar

- Frases longas demais.
- Pouca hierarquia.
- CTAs genéricos.
- Conteúdo técnico sem conexão comercial.
- Notícias antigas na Home.
- Promoções em imagem, sem texto acessível.
- Grafia inconsistente: acentos, caixa, termos técnicos.
- Falta de distinção entre fornecedor, catálogo, processo e produto.

## 3. Personalidade editorial recomendada

| Atributo | Como escrever | Evitar |
|---|---|---|
| Profissional | Frases diretas, vocabulário técnico controlado | Tom vendedor exagerado |
| Confiável | Dados concretos: Campinas, 1990, fornecedores, contato | Promessas vagas |
| Próximo | Convite claro para falar com a equipe | Linguagem fria ou robótica |
| Industrial | Termos de usinagem, processos e aplicação | Linguagem de varejo |
| Enxuto | Parágrafos curtos, headings autoexplicativos | Blocos longos sem CTA |
| Orientador | Explicar caminho: fornecedor, catálogo, orçamento | “Clique aqui” sem contexto |

## 4. Regras de escrita

### 4.1. Estrutura de parágrafo

Preferir:

```txt
1 ideia por parágrafo.
2 a 4 linhas.
Verbo de ação cedo.
Prova concreta antes de promessa.
CTA contextual no fim.
```

Evitar:

```txt
parágrafos longos;
frases com múltiplas orações;
adjetivos soltos;
jargão sem utilidade;
promessa comercial sem prova.
```

### 4.2. Estrutura de seção

Cada seção deve responder:

```txt
O que é?
Por que importa para o usuário?
Qual prova ou contexto dá confiança?
Qual é o próximo passo?
```

### 4.3. Regra de CTA

Todo CTA deve dizer exatamente o que acontece depois.

**Ruim:**

```txt
Clique aqui
Saiba mais
Ver mais
```

**Melhor:**

```txt
Solicitar orçamento
Acessar catálogo oficial
Falar com a RECOM
Consultar disponibilidade
Ver fornecedores relacionados
Pedir orientação para esta aplicação
```

### 5.2. Termos com cuidado

| Termo | Risco | Recomendação |
|---|---|---|
| Produtos | Pode parecer e-commerce | Usar quando houver item/família clara |
| Catálogo | Pode parecer catálogo próprio | Usar “catálogos oficiais dos fornecedores” |
| Promoção | Pode parecer varejo | Usar com validade, condição e CTA |
| Técnico | Pode soar restritivo | Preferir “atendimento comercial com apoio técnico”, se verdadeiro |
| Representante | Pode conflitar com posicionamento atual | Validar com proprietário; usar distribuidor quando aprovado |

## 7. Raciocínio de escrita RECOM

### 7.1. Antes de escrever, decidir a intenção do usuário

O visitante normalmente chega com uma destas intenções:

1. confirmar se a empresa existe e é confiável;
2. acessar catálogo de uma marca;
3. encontrar caminho por processo de usinagem;
4. pedir orçamento;
5. validar telefone/endereço;
6. consultar promoção;
7. entender se a RECOM atende determinada necessidade.

A redação deve começar pela intenção, não pela vontade interna de “mostrar tudo”.

### 7.2. Árvore de decisão para copy

```txt
Se a página é institucional:
  história + prova local + papel atual + contato

Se a página é fornecedor:
  nome da marca + relação com a RECOM + linhas/processos + catálogo + contato

Se a página é processo:
  aplicação + famílias relacionadas + fornecedores + orientação + contato

Se a página é promoção:
  condição + fornecedor/produto + validade + ressalvas + CTA

Se a página é contato:
  instrução simples + campos úteis + canais diretos + confirmação
```

### 7.3. Critério de qualidade

Uma boa copy da RECOM deve permitir que o usuário responda em poucos segundos:

```txt
Onde estou?
O que a RECOM faz?
Este caminho é para fornecedor, processo, promoção ou contato?
O clique me leva para catálogo externo ou para falar com a empresa?
Que informação preciso enviar para receber ajuda?
```

## 8. Guia para ghostwriter humano ou LLM

### 8.1. Prompt base para escrever como RECOM

```txt
Escreva para a RECOM Metal Duro, distribuidor B2B de ferramentas de corte para usinagem em Campinas.

Tom: profissional, direto, confiável, industrial e humano.
Objetivo: orientar compradores industriais a encontrar fornecedores, catálogos, processos e contato comercial.
Evite: linguagem de varejo, exagero promocional, jargão sem explicação, promessa sem prova e CTAs genéricos.
Preserve: histórico desde 1990, presença em Campinas, atendimento próximo, fornecedores reconhecidos, clareza sobre catálogos oficiais e foco em orçamento/contato.
Estrutura: título claro, parágrafo curto, prova/contexto e CTA específico.
```

### 8.2. Prompt para reescrever página

```txt
Reescreva a seção abaixo para o novo site da RECOM.

Objetivo da página:
[preencher]

Público:
comprador industrial, usuário técnico, cliente recorrente ou visitante local.

Regras:
- manter tom B2B, sóbrio e humano;
- reduzir frases longas;
- preservar informação factual;
- não inventar certificações, marcas ou autorização;
- usar CTA específico;
- diferenciar catálogo oficial de fornecedor e atendimento da RECOM;
- evitar "clique aqui", "saiba mais" e linguagem de varejo.

Texto base:
[colar texto]
```

### 8.3. Prompt para gerar microcopy de formulário

```txt
Gere microcopy para formulário de orçamento da RECOM.

Campos:
nome, empresa, e-mail, telefone/WhatsApp, fornecedor de interesse, processo de interesse, código do item, mensagem e anexo opcional.

Tom:
profissional, claro, útil e sem burocracia.

Objetivo:
ajudar o usuário a enviar contexto suficiente para a equipe retornar com precisão.

Inclua:
labels, texto de ajuda, mensagens de erro, estado de envio, sucesso e falha.
```


## 10. Padrões de escrita por página

### 10.1. Home

**Objetivo:** explicar rápido e direcionar.

**Estrutura:**

```txt
Título com proposta.
Subtítulo com papel da RECOM.
CTA primário.
Fornecedores.
Processos.
Prova de confiança.
Contato.
```

**Não fazer:**

- abrir com notícia antiga;
- usar carrossel automático como conteúdo principal;
- colocar muito texto técnico na primeira dobra.

---

### 10.2. A RECOM

**Objetivo:** confiança.

**Estrutura:**

```txt
Desde quando existe.
Onde atua.
O que distribui/organiza.
Como atende.
Por que o cliente pode falar com segurança.
```

**Não fazer:**

- texto longo sem quebras;
- prometer “liderança” sem prova;
- excesso de história sem conexão com contato.

---

### 10.3. Fornecedores & Catálogos

**Objetivo:** descoberta por marca.

**Estrutura:**

```txt
Explicar que a página organiza fornecedores.
Listar cards.
Indicar catálogo oficial.
Oferecer contato se o usuário precisar de ajuda.
```

**Não fazer:**

- card só com logo;
- botão “saiba mais” sem destino claro;
- link externo sem aviso.

---

### 10.4. Processos

**Objetivo:** descoberta por aplicação.

**Estrutura:**

```txt
Nome do processo.
Descrição curta.
Famílias relacionadas.
Fornecedores associados.
CTA de orientação ou orçamento.
```

**Não fazer:**

- virar aula extensa;
- esconder fornecedores;
- usar termos técnicos sem função prática.

---

### 10.5. Promoções

**Objetivo:** condição comercial sem varejização.

**Estrutura:**

```txt
Nome da promoção.
Fornecedor/linha.
Condição.
Validade.
Ressalvas.
CTA.
```

**Não fazer:**

- preço/condição apenas em imagem;
- promoção sem validade;
- urgência artificial.

---

### 10.6. Contato

**Objetivo:** converter demanda.

**Estrutura:**

```txt
Instrução curta.
Formulário.
Canais diretos.
Endereço/mapa.
Mensagem de confiança.
```

**Não fazer:**

- placeholders no lugar de labels;
- pedir campos demais como obrigatórios;
- deixar o usuário sem confirmação.

## 11. Exemplos de reescrita

### 11.1. Institucional

**Novo estilo:**

```txt
Fundada em 1990, a RECOM Metal Duro atua em Campinas com foco em ferramentas de corte para usinagem.

Ao longo da sua trajetória, a empresa construiu relacionamento com fornecedores reconhecidos e passou a atender clientes industriais que precisam de orientação, catálogo e contato comercial confiável.

Hoje, o site da RECOM organiza fornecedores, processos e canais de atendimento para facilitar a solicitação de orçamento e a consulta a materiais oficiais.
```

## 12. Do / Don’t

### Faça

- Escreva com clareza.
- Comece pela tarefa do usuário.
- Use prova concreta.
- Explique o destino do clique.
- Use CTAs específicos.
- Preserve a tradição, mas com linguagem atual.
- Dê contexto antes de mandar o usuário para catálogo externo.

### Não faça

- Não invente autorização, certificação ou estoque.
- Não chame tudo de “produto”.
- Não use “clique aqui”.
- Não esconda contato.
- Não coloque texto importante só em imagem.
- Não transforme a RECOM em e-commerce se o projeto é hub B2B.
- Não escrever como multinacional fabricante se a marca é distribuidora local/regional.

## 13. Critérios de aceite para novas copys

Uma copy da RECOM está aprovada quando:

- [ ] tem título específico;
- [ ] explica o contexto em até 2 parágrafos curtos;
- [ ] usa termo correto: fornecedor, processo, catálogo, promoção ou contato;
- [ ] contém CTA específico;
- [ ] não inventa fatos;
- [ ] é compatível com o posicionamento de distribuidor B2B;
- [ ] pode ser entendida por comprador industrial sem esforço;
- [ ] não depende de imagem para transmitir informação essencial;
- [ ] mantém tom sóbrio, humano e confiável.
